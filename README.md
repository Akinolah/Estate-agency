# EstateFindr - Firebase Studio Project

This is a Next.js real estate application starter generated by Firebase Studio.

## Features

- Property Listings Display & Filtering
- AI-Powered Property Recommendations (Requires Google AI API Key)
- Interactive Maps (Requires Google Maps API Key)
- Virtual Tours Section
- Mortgage Calculator
- Customer Reviews Carousel
- Contact Form
- Gallery Room
- Responsive Design

## Getting Started

1.  **Clone the repository:**
    ```bash
    git clone <your-repository-url>
    cd <your-project-directory>
    ```

2.  **Install dependencies:**
    ```bash
    npm install
    # or
    yarn install
    # or
    pnpm install
    ```

3.  **Set up Environment Variables:**

    -   Rename the `.env.example` file (if present) or create a new file named `.env.local` in the root of your project.
    -   Add the following environment variables to your `.env.local` file:

        ```dotenv
        # Google Generative AI API Key (Required for AI Recommendations)
        # Obtain a key from Google AI Studio: https://aistudio.google.com/app/apikey
        GOOGLE_GENAI_API_KEY=YOUR_API_KEY_HERE

        # Google Maps API Key (Required for Interactive Maps)
        # Obtain a key from Google Cloud Console: https://console.cloud.google.com/google/maps-apis/credentials
        # Remember to enable "Maps JavaScript API" and "Places API" for the key.
        # Restrict the key to your domain for security.
        NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=YOUR_MAPS_API_KEY_HERE

        # Optional: Google Maps Map ID (For custom map styling)
        # Create a Map ID in Google Cloud Console: https://console.cloud.google.com/google/maps-apis/studio/maps
        # NEXT_PUBLIC_GOOGLE_MAPS_MAP_ID=YOUR_MAP_ID_HERE
        ```

    -   **Important:** Replace `YOUR_API_KEY_HERE` and `YOUR_MAPS_API_KEY_HERE` with your actual API keys. Keep your `.env.local` file private and do not commit it to version control.

4.  **Run the development server:**
    ```bash
    npm run dev
    # or
    yarn dev
    # or
    pnpm dev
    ```

5.  **Run the Genkit development flow server (for AI features):**
    Open a *separate terminal* and run:
    ```bash
    npm run genkit:watch
    # or
    yarn genkit:watch
    # or
    pnpm genkit:watch
    ```
    This starts the Genkit development server, making the AI flows available to the Next.js app.

6.  Open [http://localhost:9002](http://localhost:9002) (or the specified port) with your browser to see the result.

## Project Structure

-   `src/app/`: Contains the main application pages using Next.js App Router.
-   `src/components/`: Reusable UI components.
    -   `src/components/ui/`: Components from shadcn/ui.
    -   `src/components/layout/`: Header and Footer components.
    -   `src/components/providers/`: Context providers (Maps, QueryClient).
-   `src/ai/`: Genkit AI configuration and flows.
    -   `src/ai/ai-instance.ts`: Genkit initialization.
    -   `src/ai/flows/`: AI flow definitions (e.g., `property-recommendation.ts`).
-   `src/lib/`: Utility functions (e.g., `cn` for class names).
-   `src/hooks/`: Custom React hooks (e.g., `useToast`, `use-mobile`).
-   `src/services/`: Business logic services (e.g., `mortgage-calculator.ts`).
-   `src/types/`: TypeScript type definitions (e.g., `property.ts`).
-   `public/`: Static assets.
-   `styles/`: Global styles (`globals.css`).

## Customization

-   **Styling:** Modify `src/app/globals.css` to adjust the Tailwind CSS theme and shadcn/ui variables (colors, radius, etc.).
-   **Content:** Replace placeholder data in page components (e.g., `src/app/page.tsx`, `src/app/listings/page.tsx`) with your actual data source (API, database).
-   **AI Prompts:** Refine the prompts in `src/ai/flows/` to better suit your specific recommendation logic.
-   **API Keys:** Ensure you have valid API keys set up in your `.env.local` file for Google AI and Google Maps.
